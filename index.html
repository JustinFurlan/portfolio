APPLE CLINIC · 01 / 04
Infrastruktur & DevOps
Docker · PostgreSQL · Nginx · Raspberry Pi
DockerPostgreSQL NginxRaspberry Pi

Die Basis des gesamten Projekts. Alle Dienste laufen in Docker-Containern und werden über ein einziges docker compose up gestartet — kein manuelles Einrichten, kein "funktioniert nur auf meinem Rechner".

PostgreSQL dient als Datenbank mit einem vollständigen Schema aus 8 Tabellen inklusive Foreign Keys, UUID-Primärschlüsseln und automatischen Timestamps. Die Initialisierung läuft über ein SQL-Migrationsskript das beim ersten Start automatisch ausgeführt wird.

Nginx fungiert als Reverse Proxy und leitet eingehende Anfragen an die richtigen Dienste weiter: / zur Kunden-Website, /dashboard zum internen Dashboard und /api/ zur REST-API — alles über einen einzigen Port 80.

APPLE CLINIC · 02 / 04
REST-API
Node.js · Express · JWT-Authentifizierung
Node.jsExpress JWTbcrypt

Das Herzstück der Anwendung — eine REST-API die alle Geschäftslogik enthält und als einzige Schnittstelle zwischen Frontend und Datenbank fungiert. Beide Frontends kommunizieren ausschließlich über diese API.

Die Authentifizierung läuft über JSON Web Tokens. Beim Login erhält der Mitarbeiter einen Token der bei jedem nachfolgenden Request mitgeschickt wird. Geschützte Routen prüfen diesen Token und blockieren unbefugte Zugriffe. Passwörter werden mit bcrypt gehasht und nie im Klartext gespeichert.

Die API unterscheidet zwischen öffentlichen Endpunkten (Auftrag erstellen, Status per Tracking-Code prüfen) und geschützten Endpunkten (Aufträge verwalten, Status ändern) — mit rollenbasiertem System für Admin und Techniker.

APPLE CLINIC · 03 / 04
Internes Dashboard
React · Mitarbeiterverwaltung · Auftragsverwaltung
ReactReact Router AxiosContext API

Das interne Tool für Mitarbeiter der Apple Clinic. Nach dem Login mit JWT-gesichertem Account sieht man eine Übersichtstabelle aller eingehenden Reparaturaufträge — mit Kundenname, Gerät, aktuellem Status und Erstellungsdatum.

In der Detailansicht sind alle relevanten Informationen auf einen Blick sichtbar: vollständige Kundendaten (Name, E-Mail, Telefon, Adresse), das eingeschickte Gerät, alle gemeldeten Schäden mit Einzelpreisen und der aktuelle Status. Der Techniker kann den Status ändern und einen Kommentar hinterlassen.

Jede Statusänderung wird automatisch in der Statushistorie protokolliert — wer hat wann was geändert und warum. Das sorgt für vollständige Transparenz intern.

APPLE CLINIC · 04 / 04
Kunden-Website
React · Bestellformular · Auftragsverfolgung
ReactUI/UX Mehrstufiges Formular

Die öffentliche Seite der Apple Clinic. Aufgebaut als mehrstufiges Formular das den Nutzer Schritt für Schritt durch den Bestellprozess führt — ohne dass man sich registrieren oder einloggen muss.

Im ersten Schritt wählt der Kunde sein iPhone-Modell und die Farbe. Dann wählt er aus einer Liste von Schadensarten (Display, Akku, Kamera, Ladeport, Wasserschaden) — mehrere Schäden gleichzeitig möglich, der Gesamtpreis wird live berechnet. Im dritten Schritt gibt er seine Kontaktdaten ein. Nach dem Absenden erscheint ein einzigartiger Tracking-Code.

Auf der Tracking-Seite kann jeder Kunde jederzeit den aktuellen Status seiner Reparatur nachschlagen. Der Status spiegelt in Echtzeit wider was im internen Dashboard eingetragen wurde.
